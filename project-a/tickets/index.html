HOW THIS WORKS

Each event has a venue, a date, and one or more ticket tiers (General, VIP, Early Bird). Tickets are sold as part of an order; you can hold several tickets in one order. Availability is computed live: capacity minus confirmed tickets.

API CONTRACT

GET /api/v1/events/
  ?q=&page=&page_size=&sort=
  → { count, page, results: [Event] }

GET /api/v1/events/{id}
  → EventDetail (with tiers + availability)

Event {
  id, title, starts_at, venue { city, name },
  min_price, available, total_capacity
}
